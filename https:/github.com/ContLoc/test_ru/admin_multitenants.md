Управление арендаторами
====================

В этом разделе описаны шаги, необходимые для добавления нового :ref:`tenant <tenant>,` а также представлен обзор элементов управления арендаторами платформы. Страница арендаторов — это первый экран, который вы видите при входе в платформу.

Добавление нового арендатора
-------------------------

.. внимание::

	Только владелец системы может добавить нового арендатора.

	Вам необходимо создать арендатора, чтобы иметь возможность добавлять к нему :ref:`new_workspace <new_workspace>` и PostgreSQL :ref:`instances <instance_adding`.

Чтобы добавить нового арендатора:

\#. На странице арендаторов нажмите «Добавить арендатора».

	.. thumbnail:: ../_static/tenants/add_tenant.png

\#. В открывшемся модальном окне выберите OperDB из списка, введите название арендатора в поле «Tenant Name» (Название арендатора) и нажмите «Create» (Создать).

	.. thumbnail:: ../_static/tenants/new_tenant.png
		:width: 300
		:align: center


После выполнения указанных шагов на странице появится новая строка с арендатором, в пределах которой вы можете создавать рабочие пространства.

Контроль арендаторов
-----------------------------

Давайте рассмотрим элементы управления в соответствии с нумерацией на рисунке ниже:

.. thumbnail:: ../\_static/tenants/tenants\_panel.png

1. Поле поиска для поиска арендаторов по имени.
2. Значок шестеренки, с помощью которого можно исправлять, удалять или изменять положение столбцов в таблице.

	.. thumbnail:: ../\_static/tenants/tenants\_list\_setting.png :width: 300 :align: center

3. Информация об арендаторах:

	* «Имя арендатора» — имя арендатора.
	* «Рабочие пространства» — количество рабочих пространств в пределах арендатора.
	* «Кластеры» — количество кластеров в пределах арендатора.
	* «Экземпляры» — количество экземпляров в пределах арендатора.
	* «Оповещения» — количество оповещений, полученных для арендатора.
	* «Неудачные задачи» — количество задач, созданных на арендаторе, которые завершились неудачей.

4. Меню арендатора со следующими опциями:

	* «Открыть арендатора» — открывает страницу с рабочими пространствами арендатора. На вкладке «Экземпляры» этой страницы можно просмотреть все экземпляры рабочих пространств арендатора.
	* «Переименовать арендатора» — открывает модальное окно для переименования арендатора. Введите новое имя арендатора в текстовое поле и нажмите «Сохранить».

		.. thumbnail:: ../\_static/tenants/tenants\_rename.png :width: 300 :align: center


	* «Дополнительная информация» — открывает страницу с расширенной информацией об арендаторе.
	* «Удалить арендатора» — открывает модальное окно для удаления арендатора. Введите «delete» в текстовое поле и нажмите «Delete».

		.. thumbnail:: ../\_static/tenants/tenants\_delete.png :width: 300 :align: center

		.. примечание::

			Перед удалением арендатора удалите все рабочие пространства в нем. Вы не можете удалить арендатора, содержащего рабочие пространства.

			.. thumbnail:: ../_static/tenants/tenants_warning.png
				:width: 300
				:align: center

Страница с дополнительной информацией об арендаторе
-----------------------------------------

На странице «Дополнительная информация» вы можете управлять правами доступа пользователей для арендатора.

Давайте просмотрим информацию на странице:

.. thumbnail:: ../\_static/tenants/info.png

1. Дополнительная информация об арендаторе:

	* количество арендаторов-владельцев;
	* количество локальных групп, созданных в рамках арендатора;
	* количество внешних групп, синхронизированных с арендатором;
	* количество локальных пользователей арендатора;
	* количество внешних пользователей арендатора.

2. Поле поиска для поиска владельцев арендаторов по имени или адресу электронной почты.
3. Кнопка для добавления владельца арендатора, при нажатии которой открывается модальное окно для добавления владельца арендатора.

	.. thumbnail:: ../\_static/tenants/new\_owner.png :width: 350 :align: center

	Чтобы добавить нового владельца, выполните следующие действия:

		* Введите имя пользователя, которому будут предоставлены права владельца арендатора.
		* Введите адрес электронной почты пользователя.
		* Введите адрес электронной почты еще раз.
		* Нажмите «Добавить владельца».

4. Список арендаторов и информация о них:

	* Имя арендатора-владельца;
	* Электронная почта арендатора-владельца;
	* дата и время выдачи пользователю прав владельца Tennant.
	* электронная почта пользователя, предоставившего права владельца для арендатора.
	* дата и время последнего входа пользователя в систему арендатора.

5. Меню владельца имеет два варианта:

	* «Отменить права» — удалить пользователя из списка владельцев арендаторов. Для этого введите «revoke» в текстовое поле открывшегося модального окна и нажмите «Revoke rights» (Отменить права).

		.. thumbnail:: ../\_static/tenants/delete\_owner.png :width: 350 :align: center

		.. примечание::
	
			Если вы отмените права пользователя, он будет удален только из списка владельцев арендатора, но сохранит права пользователя. Он сможет войти в систему арендатора, но будет видеть только те рабочие пространства, к которым у него есть доступ.

			Если вам необходимо удалить пользователя из арендатора, выполните следующие действия:

				* выберите «Отменить права» в меню «Владелец»;
				* перейдите в :ref:`список пользователей <users_common>` арендатора и :ref:`деактивируйте  <users_deactivation>` пользователя;
				* :ref:`delete  <users_delete>` user.

	* «Обновить пароль», чтобы сбросить старый пароль владельца и автоматически сгенерировать новый или выбрать свой собственный новый пароль.

		* Чтобы автоматически сгенерировать пароль, нажмите «Сбросить».

		.. thumbnail:: ../\_static/tenants/reset\_password\_1.png :width: 300 :align: center

		* Чтобы создать новый пароль вручную, снимите флажок «Автоматически генерировать пароль», введите новый пароль и нажмите «Сбросить».

		.. thumbnail:: ../\_static/tenants/reset\_password\_2.png :width: 300 :align: center
