Мониторинг резервного копирования
==================

Чтобы отобразить список резервных копий, настройте:ref: ` соединение с системой резервного копирования <backup>`.

Вы можете просмотреть список резервных копий для арендатора, рабочего пространства или конкретного экземпляра. Вы можете получить доступ к «Мониторингу резервного копирования» разными способами:

	#. В меню левой панели выберите «Модули» -> «Мониторинг резервного копирования». В этом случае откроется список резервных копий арендаторов.

		.. thumbnail:: ../_static/backup/go_to_backup_monit_1.png

	#. На странице рабочих пространств нажмите «Неудачные резервные копии». В этом случае резервные копии на странице будут отсортированы по рабочему пространству и статусу «Неудачные».

		.. thumbnail:: ../_static/backup/go_to_backup_monit_2.png

	#. В меню экземпляра выберите «Мониторинг резервного копирования». В этом случае откроется список резервных копий экземпляров.

		.. thumbnail:: ../_static/backup/go_to_backup_monit_3.png

На странице «Мониторинг резервных копий» находится список резервных копий, информация о них и элементы управления:

.. thumbnail:: ../\_static/backup/backup\_monit\_info.png

\#. Поле поиска для поиска резервных копий по FQDN. #. Кнопка «Обновить страницу». #. Резервные копии можно сортировать по рабочим пространствам, экземплярам и статусам. #. Значок шестеренки: 

	.. thumbnail:: ../_static/backup/backup_monit_setting.png
		:ширина: 300
		:align: центр

	1. С помощью этой кнопки можно изменить положение столбцов в таблице.
	2. Нажмите на значок кнопки, чтобы зафиксировать столбец в таблице.
	3. Отметьте столбцы, которые необходимо отобразить в таблице.

\#. Список резервных копий с информацией о них:

	FQDN/IP — полное доменное имя или IP-адрес экземпляра базы данных;
	Рабочее пространство — название рабочего пространства, для которого создается резервная копия;
	* Количество полных резервных копий — количество полных резервных копий;
	* Количество инкрементных резервных копий — количество инкрементных резервных копий;
	* Дата последнего резервного копирования — дата и время создания резервной копии;
	* Размер последней резервной копии;
	* Статус последней резервной копии — выполняется, завершена, не удалась или запланирована.
